# M3.5 — Frontend Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the design language, component tiers, data layer, and auth spine, then the first screen an employee uses every morning — sign in, clock in/out, and read the month's punch ledger.

**Architecture:** One client-rendered Next app. `(auth)/login` is public; `(app)/*` sits behind a `SessionProvider` + `QueryClientProvider` and a token guard. A `localStorage` bearer token is attached by `lib/api.ts` and cleared on any 401. `useSession()` (one `GET /me`) is the single source of role/scope truth. Carbon tokens live in one hand-written `carbon.css`; components are tiered, and only what the screens consume gets built.

**Tech Stack:** Next 16 (App Router) · React 19 · TypeScript · Tailwind v4 · TanStack Query 5 · Vitest + Testing Library · IBM Plex Sans (self-hosted via `next/font/local`)

**Spec:** `docs/superpowers/specs/2026-07-24-m3.5-frontend-foundation-design.md` — read it first.

## Global Constraints

- **Next 16 / React 19 / Tailwind v4 / TanStack Query 5 — pinned.** No version bumps inside the milestone.
- **`DESIGN.md` (repo root) is the token authority; `src/styles/carbon.css` is the one place tokens enter code.** Every colour, type step, and spacing value derives from it. `--radius: 0px`.
- **IBM Plex Sans is self-hosted** via `next/font/local` with vendored `woff2` (weights 300/400/600). No runtime CDN request. The font is SIL OFL — include its licence file.
- **Calendar dates on the wire are `YYYY-MM-DD` strings, never `Date` objects.** A punch at 00:30 Asia/Manila belongs to the 30th; a browser in another zone must not disagree. `lib/date.ts` does month arithmetic on strings.
- **Worked time is integer minutes**; `lib/duration.ts` (already present) is the only formatter. Never a float.
- **Success is `{data}`, errors are `{error:{code,message,details}}`.** `lib/api.ts` is the only module that knows this. Branch on `code`, never on `message` or a raw status.
- **Keys come from `lib/keys.ts`, never a string literal.** Invalidation is a typed prefix.
- **`useSession()` is the only source of role/scope truth.** Never a second `GET /me`.
- **Client guards are UX; the server is the authority.** Hiding a control is convenience — the policy already refused it.
- **The punch mutation sends `Idempotency-Key` (`crypto.randomUUID()`), reusing the same key on retry** of the same attempt.
- **Login failure copy is identical for a wrong password and an unknown email** — "That email and password don't match." M2 made those indistinguishable deliberately (constant-time hashing); the UI must not leak what the API refuses to.
- **`tsconfig` traps:** `erasableSyntaxOnly` (no constructor parameter properties, no `enum`), `verbatimModuleSyntax` (type-only imports MUST use `import type`), `noUnusedLocals`/`noUnusedParameters`, `strict`. Type errors surface via `npm run typecheck` (tsgo), not the Next build (`ignoreBuildErrors: true`).
- **Product name lives in exactly one constant** (`src/lib/brand.ts`, `PRODUCT_NAME`). Default `'HRIS'`. It is a one-line change — do not scatter it.
- **Gate:** `npm run lint && npm test && npm run typecheck && npm run build` all green. Backend suite (267) must remain untouched and green.

Key: `hris.token`. **SSR-safe:** every function no-ops / returns `null` when `typeof window === 'undefined'`.
  - `api.ts` — unchanged envelope behaviour, plus: attaches `Authorization: Bearer <token>` when a token exists; on a `401` response calls `clearToken()` then `emitLogout()` before throwing the `ApiError`. Adds typed wire calls (below).

```ts
// wire types — verified against SessionResource.php and AttendanceLogResource.php
export type PunchDirection = 'in' | 'out'
export type PunchSource = 'web' | 'manual' | 'device' | 'adjustment'
export type PunchVerification = 'verified' | 'flagged'

export type AttendanceLog = {
  id: string
  employee_id: string
  office_id: string
  punched_at: string           // ISO8601 WITH offset
  direction: PunchDirection
  source: PunchSource
  verification: PunchVerification
  flag_reason: string | null
}

/** Keyed by office-local YYYY-MM-DD — the grouping AttendanceMonth::group produces. */
export type AttendanceMonth = Record<string, AttendanceLog[]>

export type Session = {
  user: { id: string; email: string; name: string }
  employee: {
    id: string
    employee_no: string
    current_office_id: string | null
    current_department_id: string | null
  } | null
  is_system_admin: boolean
  has_reports: boolean
  hr_offices: string[]
  permissions: string[]
}

export const api = {
  health: () => request<Health>('/health'),
  login: (email: string, password: string) =>
    request<{ token: string; user: Session['user'] }>('/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ email, password }),
    }),
  logout: () => request<null>('/logout', { method: 'POST' }),
  me: () => request<Session>('/me'),
  myAttendance: (month: string) => request<AttendanceMonth>(`/me/attendance?month=${month}`),
  punch: (direction: PunchDirection, idempotencyKey: string) =>
    request<AttendanceLog>('/attendance/punch', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', 'Idempotency-Key': idempotencyKey },
      body: JSON.stringify({ direction }),
    }),
}
```

**Verify `hr_offices` is an array of office id strings** by reading `BuildSession`/`SessionData` before finalising the type; correct it if it carries objects.
